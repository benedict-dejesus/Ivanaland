# 🏝️ IVANALAND

**Help recover the 100 lost iPhones hidden throughout Ivanaland.**

A mobile-first hidden-object exploration game — one seamless, living, handcrafted
cartoon world with ten districts, a hundred hiding stories, and a statue that has
been watching over the festival all along.

Created & developed by **Benedict de Jesus**.

## The story

When Ivana Alawi announced a giveaway of 100 iPhones, thousands of people made
videos, art, and posts. This project took a different road: instead of asking for
an iPhone, it *builds a world* — a creative gift for the community. No prizes, no
catch. Just the joy of looking closely and the delight of finding.

## Play

- **Tap** anything — the world answers. Phones pop, containers open, secrets reveal.
- **Drag** to explore, **pinch / scroll** to zoom, **double-tap** to dive in.
- Enter your explorer name, find all 100, and earn a shareable
  **Certificate of Completion** with your name on it.

## Features

- 🌏 One seamless world, 10 districts, zero loading screens
- 📱 100 uniquely hidden iPhones — every one has a story, a hint, and a discovery
  method (visible, camouflaged, containers, reflections, interactions, multi-step
  sequences, and 4 legendary finds)
- 🐕 A living world: patrolling shuttle, NPCs, pets, drones, boats, crowds
- 🏆 24 achievements rewarding curiosity, observation, and persistence
- ✅ Live "iPhones Found" counter + full 1–100 checklist with hints
- 🎉 High-energy discovery celebrations (starbursts, shockwaves, confetti)
- 📜 Completion certificate + share cards (PNG export, Web Share on mobile)
- 💾 Automatic local save — no accounts, no backend, no tracking
- ♿ Reduced-motion support, screen-reader-labelled UI, generous touch targets

## Tech

Vite · TypeScript (strict) · PixiJS 8 — all art drawn procedurally in code
(zero image downloads), procedural WebAudio (zero audio files). Static deploy.

```bash
npm install
npm run dev       # local dev server
npm run build     # typecheck + production build → dist/
```

Deploys to GitHub Pages via the included workflow (`.github/workflows/deploy.yml`).
